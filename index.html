Project 05
FASTA Parser & MSA Scorer
Bioinformatics pipeline for parsing genetic data and scoring pairwise sequence alignments, enabling phylogenetic analysis of mtDNA and Y-chromosome variation across individuals.
Two-part workflow

Part 1 - FastaParser.py
Reads a cp950-encoded genetic data file and writes two FASTA files, one for mtDNA and one for Y chromosome.

Part 2 - MSA.py
Reads any aligned FASTA file and scores every pairwise alignment using match, gap, transition, and transversion weights.

Scoring parameters
KeyScoreDescription
match+1Identical bases
gap-1Insertion / deletion
transition-1A to G or C to T
transversion-2A/G to C/T

Project 06 · MSc Thesis · Lund University 2025
OriGENE: Cancer Driver Gene Classification
Benchmarking deep CNNs for cancer driver gene prediction by combining H3K4me3 epigenetic signals with mutation-guided attention to classify oncogenes and tumor suppressor genes. Code remains under prepublication review.
The Problem

Traditional cancer driver gene models often rely only on DNA mutations. Many driver genes are also epigenetically dysregulated, which makes them difficult to capture with mutation-centric classifiers alone.

Approach

OriGENE is a CNN trained on H3K4me3 ChIP-seq enrichment profiles near gene promoters. The extended version adds a mutation-guided attention branch so mutation features modulate the epigenetic feature map element-wise.

Cross-patient AUROC

Top mutation features
What the model learned

The nonsilent to silent mutation ratio remains the strongest signal, separating functionally disruptive patterns from neutral background variation.

Silent mutations per kb also remain informative, likely reflecting transcription-associated mutational processes and gene activity.

OriGENE's nonlinear attention mechanism helps capture interaction effects between mutation features and epigenetic signals that linear models often underweight.

Biomedical Provenance Review Console
An evidence-backed review console for classifying whether biomedical papers mainly use author-generated data or reused public datasets. The system combines accession evidence extraction, RAG retrieval, optional Ollama local/cloud review, and Supabase-based feedback collection.
FastAPI + JavaScript RAG-grounded LLM review Supabase feedback loop
